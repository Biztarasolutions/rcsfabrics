# Deployment Guide

## Frontend Deployment (Vercel)

### Prerequisites
- Vercel account
- Git repository

### Steps

1. **Connect Repository**
   - Go to vercel.com
   - Click "New Project"
   - Import GitHub repository
   - Select `frontend` directory as root

2. **Environment Variables**
   - Add these in Vercel dashboard:
   ```
   NEXT_PUBLIC_API_URL=https://api.yourdomain.com/api
   NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
   ```

3. **Deploy**
   ```bash
   git push origin main
   ```
   Vercel will automatically deploy on push

### Domain Setup
1. Go to Settings > Domains
2. Add custom domain
3. Update DNS records as shown
4. Wait for verification

## Backend Deployment (Railway/Render)

### Using Railway

1. **Sign Up**
   - Go to railway.app
   - Sign in with GitHub

2. **Create New Project**
   - Click "Create New Project"
   - Select "Provision PostgreSQL"
   - Select "Deploy from GitHub"
   - Select your repository

3. **Environment Variables**
   - Add in dashboard:
   ```
   NODE_ENV=production
   DATABASE_URL=<auto-generated>
   JWT_SECRET=<generate strong secret>
   FRONTEND_URL=https://yourdomain.com
   STRIPE_SECRET_KEY=sk_live_...
   ```

4. **Deploy**
   - Push to GitHub
   - Railway auto-deploys

### Using Render

1. **Sign Up**
   - Go to render.com
   - Connect GitHub

2. **Create Web Service**
   - New > Web Service
   - Connect GitHub repo
   - Select `backend` directory

3. **Configuration**
   - Runtime: Node
   - Build: `npm install && npm run build`
   - Start: `npm run start`

4. **Environment Variables**
   - Add same as above in Environment tab

5. **Database**
   - Create PostgreSQL instance
   - Update DATABASE_URL

## Database Setup (PostgreSQL)

### Using Railway Postgres
Automatically provisioned with Railway backend

### Using AWS RDS

1. **Create DB Instance**
   - Go to AWS Console
   - RDS > Create Database
   - PostgreSQL 14+
   - Free tier eligible for testing

2. **Configuration**
   - DB Instance ID: `rcsfabrics-prod`
   - Username: `postgres`
   - Password: Generate strong password
   - Initial DB: `rcsfabrics`

3. **Security**
   - Allow inbound 5432 from backend IP

4. **Connection**
   ```
   DATABASE_URL=postgresql://username:password@endpoint:5432/rcsfabrics
   ```

### Using Render Postgres
Built-in option when creating Render web service

## Troubleshooting

### Build Fails
```bash
# Clear cache and redeploy
rm -rf .next
npm run build
```

### Database Connection Error
- Verify DATABASE_URL format
- Check firewall rules
- Ensure database is running
- Test connection locally

### High Latency
- Check API response times
- Review database queries
- Enable caching
- Check API rate limiting
